## 1. 本文是干什么的

### 1.1 定位（和其他文件的区别）

| 文件 | 回答的问题 |
|------|------------|
| `跨系统功能关联关系.md` | **规则是什么**（触发条件、关联谁、为什么） |
| `_模块词典.md` | **模块叫什么**（三系统归一命名） |
| **`_跨系统语料索引.md`** | **规则/细节原文在哪**（去读哪条 MCP、哪份 PRD） |
| `系统定位/用户角色/用户场景矩阵` | **结构化知识**（边界、角色、场景） |

### 1.2 典型使用场景

| 场景 | 怎么做 |
|------|--------|
| Agent 写影响分析，CS 节只有摘要 | 按主题查 §4 → `get_requirement_detail` 读全文 |
| 新跨系统 PRD 上线，要补 CS 规则 | 读对应 PRD/MCP → 更新 CS → 在 §3/§4 补索引行 |
| 用户描述「账期」「最后一票」等 | §5 速查 → 定位 CS + MCP 版本-序号 |
| 秒账 MCP 不可用 | §3 本地 PRD P0 清单 + `data/prd` grep |

### 1.3 不是什么

- **不是**影响模块清单的替代品（模块名用 `_模块词典.md`）
- **不是**规则正文（不要从这里复制大段规则进 PRD，应蒸馏进 CS 节）

## 2. 推荐阅读工作流

```
需求/改动点
    ↓
读 跨系统功能关联关系.md（命中 CS-XX？）
    ↓ 需要细节/证据
查 本文 §5 主题速查 或 §4 MCP 主题表
    ↓
┌─────────────────┬─────────────────┐
│ 本地 PRD         │ MCP              │
│ Read data/prd/…  │ get_requirement_detail(version, index) │
└─────────────────┴─────────────────┘
    ↓
核对后更新 CS 节 / 影响分析 / 待确认项
```

**原则**：先读已蒸馏的 CS；**只有**字段级、状态机、端差异、公式不明时才下钻原文。

## 6. MCP 工具调用手册

### 6.1 检索顺序（与 `skills/_shared/mcp-retrieval.md` 一致）

```
1. search_by_keyword（需求关键词）
2. search_by_module（已知模块，见 _模块词典.md §9）
3. search_by_impact_scope（PRD 影响范围行）
4. get_requirement_detail（读全文）
5. 仍不足 → 本文 §3 本地 PRD + grep
```

### 6.2 按需求类型选服务器

| 需求类型 | 服务器 | 推荐动作 |
|----------|--------|----------|
| ERP 销售/采购/报表/商户设置 | `user-mzfileserver` | `search_by_module`；异常则本地 PRD |
| 云仓出/入库、秒账↔WMS | **双库** | 本地 V4.9.0 + wms `search_by_keyword`: 秒账 |
| WMS 拣货/托盘/指令 | `user-wmsfileserver` | `module`: 出库单、指令列表、托盘详情 |
| BSS 货主/账单/签约 | `user-wmsfileserver` | `module`: 货主签约、月账单、优惠设置 |
| 跨系统规则补漏 | `user-wmsfileserver` | 关键词：同步、账期、最后一票、审批 |

### 6.3 `get_requirement_detail` 示例

```text
server: user-wmsfileserver
version: "1.3.9"
index: 1
→ 账期全文（CS-08）
```

```text
server: user-wmsfileserver
version: "1.4.5"
index: 4
→ 货主签约审批（CS-07）
```

### 6.4 结果使用原则

- MCP 原文用于**核对字段、状态、端差异、公式**；写入 CS 时须**蒸馏**，不全文粘贴
- 本地 PRD 与 MCP 冲突 → 较新版本优先；无法判断 → `待确认项`
- 负责人维度需求「（霍）…」→ 功能模块以正文「影响模块」为准，忽略负责人标签
