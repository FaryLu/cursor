# 秒账功能关联关系

> 用途：第一步影响分析时，从主功能展开候选影响范围。
> 规则：按大模块组织；只写「关联谁、为什么、何时检查」，不写直接/间接/横切分类。
> 关联项名称须与 `秒账核心功能.md` 一致。
> 详细规则见各模块知识卡片或独立文档时，在表中标注引用路径。

---

## 使用说明

1. 识别需求主功能（对照 `秒账核心功能.md` 第 1 节）。
2. 读取本文件对应章节「关联检查」+ 文末「全局补漏规则」。
3. 若涉及开单方式、产品属性、生成单据、消息提醒，再读对应横向章节。
4. 用 `秒账核心功能.md` 触发条件逐项校验，合并去重后输出影响模块。
5. 子能力列不全时，以「本模块子能力 + 补漏规则 + 历史 PRD/MCP」补充，标记 `待确认项`。

## 全局补漏规则

来源：`秒账核心功能.md` 第 0、5、6 节及历史 PRD 高频影响规律。

| 触发条件 | 必须检查的核心功能 |
|----------|-------------------|
| 新增/修改字段 | 导入、列表导出、详情导出、打印、自定义模板、字段权限、角色权限、老数据兼容 |
| 新增业务类功能 | 商户设置（业务属性开关）、新账号默认值、老数据兼容；明确新数据跟随实时商户设置还是单据商户设置 |
| 新增交互/个人偏好类 | 商户设置（偏好设置开关）、新账号默认值 |
| 修改金额、折扣、优惠、欠款、合同金额 | 收款单/付款单、报表、字段权限、角色权限、业务日志、自定义公式 |
| 修改数量、仓库、库存口径 | 库存、云仓出库单、云仓入库单、报表、业务日志；库存回滚逻辑、秒账可用/在途、云仓可用/在途 |
| 修改状态、审核、作废、草稿 | 审批功能、消息提醒、业务日志、相关生成单据链路 |
| 修改扫码、条码、SN、维度条码 | 产品、库存、扫码开单、打印 |
| 修改商户设置开关 | 老数据兼容、新账号默认值、业务日志、结账/归档（若涉及封存数据） |
| 修改单价、售价、进价取值 | 智能记录、五次单价记录、报表、字段权限 |
| 涉及账期、封存 | 结账、归档、老数据兼容 |
| 涉及分店 | 分店申请单、分店消息、账号权限、角色权限 |
| 涉及云店 | 秒账云店、云店消息、产品、库存、销售单 |
| 涉及 WMS/云仓 | 云仓出库单、云仓入库单、云仓消息、库存、云仓库存相关报表 |
| 仅改文案/弱展示，不改字段/状态/金额/数量 | 可弱化报表、消息提醒、生成单据（须在影响分析中说明原因） |

## 业务开单方式（横向）

适用于：销售单、采购单、销售退货单、采购退货单、加工单等可开单录入场景。

| 关联功能 | 关联原因 | 何时检查 |
|----------|----------|----------|
| 扫码开单 | 条码/SN/维度条码/布匹标签识别与产品带出 | 改开单入口、扫码逻辑、条码规则 |
| Ai开单 | 文本/语音/图片识别录入 | 改识别入口、识别结果映射产品行 |
| 以图搜图开单 | 图片匹配产品图开单 | 改以图搜图、产品图片维护 |
| 产品 | 扫码/识别后的产品、规格、单位带出 | 改开单选品逻辑 |

| 库存 | 开单数量与库存校验、可用库存展示 | 改数量、仓库、负库存校验 |
