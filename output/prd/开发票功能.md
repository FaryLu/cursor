# 开发票功能

1. 需求背景：
    - 商户在线下完成实际开票后，需要在秒账销售单中登记开票情况，便于业务员、财务核对「哪些单已开票、开了多少、开给了哪个抬头」。
    - 本次不对接第三方税控/电子发票平台，仅在系统内做开票台账；其他单据类型（采购单、退货单等）暂不支持。
    - 商户可在两种模式间选择：**仅维护开票状态**，或 **维护抬头 + 日期 + 金额等明细**；明细模式下同一销售单可登记多条开票记录。
    - 开票金额作为独立台账记录，**不与**销售单合同金额、已收款勾稽校验。

2. 影响模块：
    - 商户设置（业务属性、商家开票公司）
    - 销售单（列表、详情/编辑、打印、导出）
    - 客户（开票公司、导入、导出）
    - 员工管理（角色权限、账号权限）
    - 业务日志、打印、自定义模板

3. 验收标准：
    - P0：开启功能且选择「仅状态模式」后，销售单列表展示开票状态列并支持筛选；详情/编辑页可修改开票状态并保存。
    - P0：选择「明细模式」后，销售单详情/编辑页可新增/编辑/删除多条开票记录（含双方抬头、开票日期、开票金额）；列表开票状态随明细汇总正确展示。
    - P0：商户设置可维护多个商家开票公司；客户资料可维护多个客户开票公司；明细模式登记时可选择带出。
    - P0：销售单打印输出包含开票状态；明细模式下打印包含开票记录明细（或汇总+明细，见打印区规则）。
    - P0：无权限账号不可查看/编辑开票信息；关键操作记入业务日志。
    - P1：客户开票公司支持导入；销售单/客户导出字段包含新增开票相关字段。
    - P2：销售单自定义打印模板支持开票相关占位符。

## 1 商户设置

### 1.1 业务属性

#### 1.1 开票功能配置区

1. 开票功能开关：
    - 交互逻辑：商户设置-行业信息-业务属性新增「开票功能」开关；默认关闭；新账号默认关闭。关闭时销售单不展示开票相关入口，历史已登记数据只读展示（老数据兼容见数据规则）。
    - 操作权限：复用商户设置-业务属性编辑权限，参考 `V5.3.0审批功能` 中增值服务/业务属性开关配置交互。

2. 开票模式：
    - 交互逻辑：开启「开票功能」后显示，单选：**仅状态模式** / **明细模式**；新账号默认「仅状态模式」。切换模式仅影响切换后新建/编辑时的录入界面，已保存数据保留原样展示。
    - 操作权限：同业务属性编辑权限。

3. 模式说明（问号提示）：
    - 交互逻辑：仅状态模式——销售单仅选择开票状态；明细模式——销售单可登记多条开票记录（抬头、日期、金额等）。

#### 1.2 商家开票公司维护区

1. 开票公司列表：
    - 交互逻辑：开启「开票功能」后，在商户设置中提供「商家开票公司」维护入口（可与基本信息/辅助资料同级 Tab，具体 UI 复用现有辅助资料列表样式）。支持新增、编辑、禁用/删除（无业务引用时可删，有引用时仅禁用）。
    - 操作权限：老板及具备商户设置编辑权限的角色可操作；只读角色可查看。

2. 开票公司字段（每条）：
    - 交互逻辑：
        - 公司名称：必填，文本，长度复用现有公司名称字段规则。
        - 纳税人识别号：选填，文本。
        - 地址、电话：选填，文本。
        - 开户行及账号：选填，文本。
        - 是否默认：单选默认项，保存时至少可设一个默认（多条时仅一条默认）。
    - 字段权限：受商户设置查看/编辑权限控制。

3. 列表数据规则：
    - 排序规则：默认按创建时间倒序，默认项置顶。
    - 老数据兼容：功能首次开启时，若无商家开票公司，引导用户至少新增一条；历史销售单在明细模式下选择抬头时若原抬头已删除，展示原值且标记「抬头已失效」。

## 2 客户

### 2.1 客户详情页 / 编辑页

#### 2.1 开票信息区

1. 开票公司列表（子表）：
    - 交互逻辑：开启商户「开票功能」后，客户详情/编辑页展示「开票公司」区域；支持为客户维护**多个**开票公司。字段与商家开票公司一致（公司名称、税号、地址电话、开户行账号、是否默认）。明细模式销售单登记开票记录时，默认带出客户默认开票公司，可改选其他已维护抬头。
    - 字段权限：复用客户编辑权限；无客户编辑权限时只读。
    - 显示条件：商户已开启开票功能；否则整个区域隐藏。

2. 新增 / 编辑 / 删除开票公司：
    - 操作权限：有编辑客户权限时可操作；删除规则同商家开票公司（有开票记录引用时不可物理删除，可禁用）。
    - 交互逻辑：保存客户时一并保存开票公司子表；至少 0 条，不强制客户必须有开票公司。

#### 2.2 客户导入

1. 导入模板扩展：
    - 交互逻辑：客户导入模板增加默认开票公司名称、税号等列；同一客户多个抬头首版在界面补录，批量多抬头导入列为 P1。
    - 操作权限：复用客户导入权限。

### 2.2 客户列表页

1. 列表字段：
    - 交互逻辑：列表默认不增加开票公司列（避免列过多）；详情/导出可见。

## 3 销售单

### 3.1 列表页

#### 3.1 搜索区

1. 开票状态：
    - 交互逻辑：下拉多选，选项：**未开票**、**部分开票**、**已开票**；明细模式下三项均可用，仅状态模式下「部分开票」不可选或筛选项隐藏。
    - 字段权限：有查看销售单列表权限时显示。
    - 数据范围：筛选结果受销售单数据权限约束。

#### 3.2 列表字段

1. 开票状态（基础字段）：
    - 交互逻辑：展示当前销售单汇总开票状态；文案：未开票 / 部分开票 / 已开票。
    - 字段权限：默认固定展示；若后续增加「查看开票信息」字段权限，无权限时显示「—」或隐藏列。

#### 3.3 列表数据规则

1. 开票状态汇总规则：
    - **仅状态模式**：列表展示用户在销售单上手动选择的状态（未开票 / 已开票）。
    - **明细模式**（自动汇总，仅用于展示，不做保存校验）：
        - 未开票：无任何开票记录；
        - 部分开票：有开票记录，且开票金额合计 < 销售单合同金额；
        - 已开票：有开票记录，且开票金额合计 ≥ 销售单合同金额。
    - 合同金额仅用于上述展示分档，**不参与保存校验**；开票金额与已收款互不挂钩。

2. 数据权限：
    - 列表结果集复用现有销售单列表数据权限（查看自己 / 查看全部 / 分店范围等）。

### 3.2 详情页 / 编辑页

#### 3.1 开票信息区（仅状态模式）

1. 开票状态：
    - 交互逻辑：下拉单选，**未开票** / **已开票**；保存随销售单一并提交。
    - 字段权限：有编辑销售单权限且具备「编辑销售单开票信息」操作权限时可编辑；详情页只读展示。
    - 编辑权限：草稿/已保存单据可编辑规则复用现有销售单编辑规则（审批中/已归档等限制复用现有单据状态规则）。

#### 3.2 开票记录区（明细模式）

1. 开票记录列表：
    - 交互逻辑：表格展示当前销售单下多条开票记录，列：商家开票公司、客户开票公司、开票日期、开票金额、备注（选填）、操作（编辑/删除）。支持「新增开票记录」按钮打开弹框或行内编辑。
    - 操作权限：「新增/编辑/删除开票记录」受角色操作权限控制。
    - 字段权限：无「查看开票信息」权限时整个区域隐藏。

2. 商家开票公司：
    - 交互逻辑：下拉单选，数据源=商户设置中启用的商家开票公司；默认选中默认商家抬头。
    - 字段权限：有编辑权限时可改。
    - 数据范围：当前商户下全部启用中的商家开票公司。

3. 客户开票公司：
    - 交互逻辑：下拉单选，数据源为当前销售单客户下启用的客户开票公司，默认带出客户默认抬头；客户未维护抬头时，保存该条记录提示「请先在客户资料维护开票公司」。
    - 数据范围：当前销售单所选客户下的开票公司。

4. 开票日期：
    - 交互逻辑：日期选择器，必填，默认当天。
    - 输入格式：日期。

5. 开票金额：
    - 交互逻辑：金额输入，必填，正数，精度复用系统金额精度（2 位小数）；不与合同金额、已收款比较校验。
    - 字段权限：可配置独立「开票金额」字段查看权限；无权限时隐藏金额。

6. 备注：
    - 交互逻辑：选填，文本，长度复用现有备注字段规则。

7. 新增 / 编辑 / 删除开票记录：
    - 操作权限：独立操作权限「维护销售单开票记录」；无权限时仅可查看。
    - 交互逻辑：删除需二次确认；删除后重算列表开票状态。同一销售单允许 0~N 条记录。

#### 3.3 页面数据规则

1. 金额独立台账：
    - 开票金额合计不写入已收款、不影响欠款、不触发收款消息、不回写合同金额。

2. 老数据兼容：
    - 功能关闭前已存在的销售单无开票数据；功能开启后可对历史单补录。
    - 模式从明细切到仅状态：已有明细记录只读保留，仅状态字段可编辑。

### 3.3 打印

#### 3.1 打印内容

1. 开票状态 / 开票记录：
    - 交互逻辑：商户开启开票功能且销售单打印时——**仅状态模式**：打印件展示「开票状态：未开票/已开票」；**明细模式**：打印件展示开票记录明细表（商家抬头、客户抬头、开票日期、开票金额、备注）及合计开票金额。
    - 操作权限：复用销售单打印权限。
    - 字段权限：无查看开票信息权限时，打印件不输出开票区域（或输出「—」）。

2. 商户设置-打印信息：
    - 交互逻辑：销售单打印信息设置中增加开票相关字段勾选（是否打印开票区域、明细模式是否打印合计）；复用 `V3.6.5 打印完善` / 自定义模板字段勾选逻辑。

### 3.4 导出

1. 列表导出 / 详情导出：
    - 交互逻辑：列表导出增加「开票状态」「开票金额合计」列；详情导出含开票记录明细（明细模式）。
    - 操作权限：复用销售单导出权限。

## 4 员工管理

### 4.1 角色权限

#### 4.1 销售单开票权限

1. 查看销售单开票信息：
    - 操作权限：角色可勾选；无权限时销售单列表开票状态列、详情开票区域隐藏或脱敏。
    - 交互逻辑：配置入口在员工管理-角色管理-权限树，挂载在「销售单」节点下。

2. 编辑销售单开票信息（仅状态模式改状态）：
    - 操作权限：独立操作权限；依赖「编辑销售单」权限，子集关系复用现有「编辑销售单但未收款」等粒度设计。

3. 维护销售单开票记录（明细模式增删改记录）：
    - 操作权限：独立操作权限；可单独授予财务角色。

4. 开票金额字段权限：
    - 字段权限：明细模式下「开票金额」可单独控制查看；无权限时不展示金额数值。

#### 4.2 客户 / 商户设置权限

1. 维护客户开票公司：
    - 操作权限：复用「编辑客户」权限。

2. 维护商家开票公司：
    - 操作权限：复用商户设置编辑权限。

#### 4.3 账号权限

1. 数据范围：
    - 数据规则：开票信息随销售单数据权限——仅能查看/编辑有权查看的销售单上的开票记录；不单独做开票数据范围。

### 4.2 业务日志

1. 记录范围：
    - 交互逻辑：记录销售单开票状态变更；明细模式下记录开票记录的新增、编辑、删除（含金额、日期、抬头变更）；记录商家/客户开票公司的新增、编辑、禁用。
    - 操作权限：日志查看权限复用现有业务日志规则。

## 5 关联说明（横切）

### 5.1 自定义模板

1. 销售单自定义打印模板：
    - 交互逻辑：新增占位符：开票状态、开票金额合计、开票记录明细表（明细模式）；复用 `V4.8.5 自定义模板` 占位符机制。

### 5.2 明确不做

1. 第三方发票平台对接、电子发票开具、发票验真。
2. 采购单、退货单、收款单、报表统计（后续版本可扩展）。
3. 开票金额与合同金额、已收款的勾稽、超开/少开校验。
