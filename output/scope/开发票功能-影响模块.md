# 开发票功能 — 影响模块（已确认）

## 需求摘要

- **主功能**：销售单
- **需求类型**：新增功能（销售单维度开票台账）
- **边界**：
  - 仅销售单，其他单据暂不做
  - 不对接第三方发票/税控系统，用户线下开票、系统内登记
  - 维护双方开票抬头：商家开票公司、客户开票公司（均支持多个）
- **开票模式**（商户设置可选）：
  1. **仅状态模式**：销售单仅选择/展示开票状态
  2. **明细模式**：维护抬头、开票日期、开票金额等明细，同一销售单可登记多条开票记录
- **业务规则（已确认）**：
  - 开票记录含：日期、金额；明细模式含双方抬头
  - 销售单列表需开票状态列及筛选
  - 开票金额与合同金额、已收款**不挂钩**，纯台账
  - 销售单打印需带出开票信息

---

## 影响模块

### 一、直接影响

- **销售单**
  - 子模块：列表页（开票状态列、筛选）、详情页/编辑页（开票状态或开票记录区）
  - 关联理由：核心挂载点；两种模式下列表均展示开票状态，明细模式在详情/编辑维护多条开票记录
  - 来源：业务规则推导 + 用户确认

- **商户设置**
  - 子模块：业务属性（开票功能开关、开票模式选择）、商家开票公司维护
  - 关联理由：功能开关、模式配置、商家侧抬头主数据
  - 来源：关联关系 + 核心功能校验 + 用户确认

- **客户**
  - 子模块：客户详情/编辑（开票公司信息，支持多个）、导入
  - 关联理由：客户侧开票公司主数据，明细模式开票记录选择带出
  - 来源：关联关系 + 用户确认

### 二、间接影响

- **销售单**
  - 子模块：打印、列表导出、详情导出、自定义模板
  - 关联理由：打印需带出开票信息；新增字段后导出/模板需同步评估
  - 来源：关联关系 + 用户确认

- **客户**
  - 子模块：列表导出、详情导出、导入
  - 关联理由：客户开票公司字段纳入导入导出
  - 来源：关联关系 + 核心功能校验

### 三、横切影响

- **员工管理**
  - 子模块：角色权限（字段权限、操作权限）、账号权限（数据范围）
  - 关联理由：权限配置入口在员工角色管理；需新增开票相关权限项
  - 来源：关联关系 + 用户确认

- **业务日志**
  - 子模块：开票记录/开票状态变更、开票抬头维护
  - 关联理由：开票台账为关键业务数据
  - 来源：关联关系 + 核心功能校验

- **打印**
  - 子模块：销售单 A4/热敏等打印模板、商户设置-打印信息
  - 关联理由：用户确认销售单打印需带出开票信息
  - 来源：用户确认 + 关联关系

- **老数据兼容 / 新账号默认值**
  - 子模块：功能开关关闭时历史单表现、新商户默认模式
  - 关联理由：新增业务类功能标准项
  - 来源：核心功能校验

### 四、补漏检查

- **已检查**：全局必检项、全局补漏规则、销售单/客户/商户设置/员工管理关联、核心功能清单
- **可省略项**：采购单、收款单、库存、报表、消息提醒、生成单据、秒账云店、审批（是否限制已审批单编辑开票，PRD 中按现有单据编辑规则处理）
