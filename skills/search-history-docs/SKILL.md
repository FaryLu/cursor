## 前置上下文

执行本 skill 前，必须读取：@skills/_shared/prd-context.md

## 核心目标

不是只做关键词搜索，而是基于「需求描述 + 历史 PRD + 秒账新功能影响模块模板」输出完整的影响模块分析，稳定识别新功能带来的关联功能影响。

## 搜索范围

### 1. 本地文件搜索

- `../../data/prd/*/*.md` — 历史 PRD
- `../../data/modules/秒账新功能影响模块模板.md` — 影响模块检查清单（权威）
- `../../data/modules/功能.md` — 生成单据、消息提醒等联动索引（补充）

### 2. MCP 内容搜索

若配置了 MCP，按 @skills/_shared/mcp-retrieval.md 检索 `user-mzfileserver` 历史记录。

## 搜索策略

### 关键词提取

从需求描述中提取：
1. **业务关键词**：采购、销售、库存、财务等
2. **功能关键词**：订单、报表、审批、导入等
3. **对象关键词**：商品、客户、供应商、仓库等
4. **关联关键词**：生成单据、打印、导入导出、权限、日志、提醒、兼容老数据等

### 搜索执行

按以下顺序执行，不能跳步：

1. **识别主功能**
   - 提炼主功能、核心对象、核心业务动作、上下游单据/模块
   - 明确属于「新增功能 / 规则调整 / 字段扩展 / 流程变更 / 兼容性修改」中的哪一类

2. **检索历史依据**
   - 在历史 PRD 中搜索主功能及其上下游流程
   - 优先查找：同类功能、相似单据、相同字段、相同交互、相同业务规则
   - 本地不足时，按 MCP 规则补充搜索

3. **对照影响模块模板做关联展开**
   - 必须把 `秒账新功能影响模块模板` 当作检查清单，而不是参考阅读
   - 按模板结构顺序读取：`功能定位 → 常见直接影响 → 常见间接影响 → 横切检查项 → 触发补漏规则 → 可省略条件`
   - 即使用户未直接提到，只要模板或历史 PRD 显示通常会联动，也要纳入候选影响范围
   - 若需求命中多个主功能模块，合并多个模块的关联结果并去重

4. **分层判断影响范围**
   - **直接影响**：需求主流程直接发生修改的模块
   - **间接影响**：由主流程联动、生成、引用、同步、回写、反查、状态变化导致受影响的模块
   - **横切影响**：权限、导入、导出、打印、报表、消息提醒、业务日志、老数据兼容、新账号默认值、自定义模板等共性能力
   - 每个影响模块必须说明「为什么会被影响」，并标注来源：模板关联 / 本地历史 PRD 复用 / MCP 历史记录复用 / 业务规则推导
   - 若模板中存在明显相关的关联项，不能因用户需求里未直接提及而省略

5. **输出前补漏自检**
   - 自检是否遗漏：`权限`、`打印`、`导入`、`导出`、`报表`、`消息提醒`、`业务日志`、`老数据`、`新账号`
   - 若需求涉及单据、字段、状态、金额、库存、扫码、模板、生成下游单据，必须重点检查这些横切模块
   - 若模板写有「可省略条件」，只有满足省略条件且能说明原因时，才允许不纳入输出
   - 若本地历史 PRD 和 MCP 都无法支撑关键判断，停止输出最终结论，改为先追问

### 搜索优先级

1. 精确匹配（关键词完整出现）
2. 模糊匹配（关键词部分匹配）
3. 语义匹配（相关概念扩展）

## 必须追问条件

出现以下任一情况时，不能直接输出影响模块结论，必须先追问：

- 无法识别需求最接近哪个主功能模块。
- 无法判断用户角色、核心使用场景、业务目标。
- 涉及上下游单据联动，但无法判断是否生成下游单据、是否回写来源单据、是否影响状态流转。
- 涉及字段、金额、数量、库存、价格、状态、审核，但关键规则不清晰（如取值口径、是否允许编辑、是否允许负数、是否需要兼容老数据）。
- 需求明显可能影响权限、打印、导入、导出、报表、消息提醒、业务日志，但用户未说明是否需要同步调整，且无法从模板和历史文档中明确判断。
- 需求涉及商户设置开关、新账号默认值、老数据兼容，但未说明开启关闭后的处理方式。
- 模板提示应检查多个横切模块，但无法判断其中哪些需要纳入。
- 本地历史 PRD 与 MCP 结果都不足以支撑关键业务判断。

追问后再生成：只有在核心目标、主功能模块、关键业务规则、上下游关系基本明确后，才能输出影响模块；部分细节仍不明确但可高置信推断时，须标注 `待确认项`。

## 输出格式

```markdown
## 影响模块
### 一、直接影响
- **模块名称**
  - 子模块：摘要
  - 关联理由：为什么受影响
  - 来源：模板关联 / 本地历史PRD复用 / MCP历史记录复用 / 业务规则推导

### 二、间接影响
- **模块名称**
  - 子模块：摘要
  - 关联理由：为什么受影响
  - 来源：模板关联 / 本地历史PRD复用 / MCP历史记录复用 / 业务规则推导

### 三、横切影响
- **模块名称**
  - 子模块：摘要
  - 关联理由：为什么受影响
  - 来源：模板关联 / 本地历史PRD复用 / MCP历史记录复用 / 业务规则推导

### 四、补漏检查
- 已检查：权限 / 打印 / 导入 / 导出 / 报表 / 消息提醒 / 业务日志 / 老数据 / 新账号
- 可省略项：写明省略原因；不能只省略不解释
```

## 输出文件

按 @skills/_shared/output-convention.md 写入 `output/scope/{需求名称}-影响模块.md`。

## 使用示例

**输入**：产品维度条形码功能

**输出**：

```markdown
### 影响模块
### 一、直接影响
- **商户设置**
  - 子模块：货品属性、打印信息
  - 关联理由：需要新增维度条形码开关与打印字段配置
  - 来源：模板关联 + 业务规则推导

- **产品模块**
  - 子模块：产品详情、导入
  - 关联理由：条形码字段要在产品基础资料中维护，并支持批量导入
  - 来源：本地历史PRD复用 + 业务规则推导

- **销售单**
  - 子模块：产品列表、扫码开单
  - 关联理由：销售开单需要识别维度条形码并正确带出产品
  - 来源：模板关联 + 本地历史PRD复用

### 二、间接影响
- **销售单**
  - 子模块：打印、导入
  - 关联理由：新增字段后，打印模板和导入模板需同步支持
  - 来源：模板关联

### 三、横切影响
- **打印**
  - 子模块：A4打印、热敏打印
  - 关联理由：涉及新增条形码字段时，常见打印能力需要同步评估
  - 来源：模板关联

### 四、补漏检查
- 已检查：权限 / 打印 / 导入 / 导出 / 报表 / 消息提醒 / 业务日志 / 老数据 / 新账号
- 可省略项：报表、消息提醒、业务日志暂无直接影响，可暂不展开
```
